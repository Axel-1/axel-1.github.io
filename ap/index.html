GSB Médecins : API REST avec clients mobile et Web⌗

Voici le dernier AP qui conclut cette formation en BTS SIO. Tout comme le précédent, le contexte est imposé. Il s’agit d’un système d’annuaire de médecin par pays et département. Seul un fichier SQL contenant les médecins nous est fourni. Ansi, la base de données ainsi que le modèle de données est à réaliser en plus de l’application. Le contexte impose également une architecture spécifique, à savoir un backend qui expose un webservice et deux clients différents qui consomment ce webservice. J’ai choisi d’utiliser Symfony avec API Platform pour réaliser le backend qui expose une API REST. Ensuite, pour les clients, j’ai réalisé une application mobile avec Flutter pour visualiser l’annuaire et un client web d’administration pour éditer, ajouter et supprimer les médecins dans l’annuaire.

L’application dispose des fonctionnalités suivantes :

  • API REST
  • Affichage des médecins par pays et département
  • Système de recherche
  • Application mobile Flutter pour visualiser l’annuaire
  • Client web pour l’administration : ajout, modification, suppression des médecins
  • Authentification des administrateurs

GSB Gestion Visites : Application Web MVC avec framework⌗

Avec ce troisième AP, le sujet proposé était une application Web de gestion des visites des commerciaux pharmaceutiques. Le contexte comportait une base de données déjà complète. Ainsi nous devions baser notre application sur ce modèle de données. Nous étions libres de la solution technique à utiliser. J’ai donc choisi d’utiliser le framework Spring ainsi que le moteur de template Thymeleaf.

L’application dispose des fonctionnalités suivantes :

  • Gestion des rapports de visites avec gestion des médicaments offerts comme échantillons.
  • Gestion des médecins avec leurs coordonnées
  • Affichage des médicaments
  • Authentification des commerciaux

Simple Inventory : Application Web MVC sans framework⌗

Pour ce second AP, le but était de mettre en application les différentes connaissances que nous avions apprises lors de TP sur le MVC en PHP. Ainsi, j’ai réalisé une application Web MVC en PHP natif, c’est-à-dire sans framework. Le choix du contexte était libre, j’ai décidé de réaliser un système de gestion de parc informatique.

L’application dispose des fonctionnalités suivantes :

  • Gestion de plusieurs sites
  • Sauvegarde des différentes informations sur une machine : numéro de série, modèle, nom d’hote etc…
  • Suivi de la garantie : nom, durée, date d’expiration
  • Système de supervision
