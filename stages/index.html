Stages

Null Labs⌗

Dans le cadre de ma formation de BTS SIO, j’ai réalisé un stage dans l’entreprise Null Labs, qui est une entreprise de service numérique. L’objectif était de réaliser la maintenance et l’évolution d’un projet en cours de réalisation pour un client. Null Labs est une entreprise de service numérique dirigée par Benjamin Bacot.

Le projet qui m’a été confié, est un CMS personnalisé pour la gestion d’un centre de formation de danse : l’I-F-D-S.

I-F-D-S CMS est une application Web réalisée avec Symfony et PHP. Elle dispose de multiples fonctionnalités personnalisées pour les besoins de l’I-F-D-S. Tel qu’un formulaire de contact complexe pour la demande de formation ou encore un planning dynamique. Également, comme son nom l’indique, cette application est un CMS c’est-à-dire que l’utilisateur peut créer des pages lui-même depuis une interface d’administration.

Durant ce stage j’ai réalisé différentes activités :

  • Réalisation d’un environnement de développement avec Docker
  • Migration du projet vers Symfony 6.0
  • Mise en place de la gestion de projet Agile-Scrum
  • Création de Fixtures pour l’initialisation de la base de données
  • Création d’une API REST avec API Platform pour le créateur de page du CMS
  • Télécharger le rapport de stage
  • Site Web de l’application (Bientôt disponible !)

Pinoy services⌗

J’ai réalisé dans le cadre de ma formation de BTS SIO un stage dans l’entreprise Pinoy Services qui est un magasin de produit alimentaire philippin et asiatique. L’objectif était de réaliser un site web e-commerce afin de développer l’activité de vente à distance de la société.

Le projet qui nous a été confié consiste à réaliser un site web e-commerce pour l’entreprise afin de faciliter la vente à distance. Auparavant, les clients contactaient directement la gérante via Facebook ou Whatsapp afin de passer leurs commandes.

Le cahier des charges est simple, la gérante souhaite un site e-commerce complet (paiements, calcul des frais de ports, facture, e-mail, etc…). Également, l’entreprise souhaite pouvoir afficher sur le site des infos pratiques sur son point de vente physique comme les horaires, l’adresse ainsi que les taux de changes de ses deux partenaires pour les transferts d’argent.

Nous avons donc décidé d’utiliser le CMS PrestaShop. Ce logiciel nous offre beaucoup de possibilités au niveau de la configuration et de la personnalisation, grâce aux extensions. Il est libre, open-source et sponsorisé par une entreprise française. PrestaShop offre par défaut des fonctionnalités liées à la RGPD et est utilisé par de grandes entreprises.

Durant ce stage j’ai réalisé différentes activités :

  • Installation du CMS Prestashop
  • Analyse de l’architecture technique du CMS
  • Développement d’une extension pour afficher le taux de change
  • Création d’un tableau de bord pour les commandes en PHP à l’aide de l’API REST de Prestashop
  • Rédaction d’une documentation utilisateur
